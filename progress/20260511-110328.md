## 本次目标

实现 Phase 1 的第一步：PostgreSQL migration、迁移命令和 API 数据库健康检查。

## 已完成

1. 新增初始数据库迁移。
   - `services/api/migrations/000001_initial_schema.up.sql`
   - `services/api/migrations/000001_initial_schema.down.sql`
   - 覆盖 `users`、`refresh_tokens`、`documents`、`document_contents`。
   - 启用 `pgcrypto`，使用 `gen_random_uuid()` 生成 UUID。
   - 新增 `set_updated_at()` 触发器函数和核心索引。

2. 实现轻量迁移执行器。
   - 新增 `internal/database/migrations.go`。
   - 使用 `schema_migrations` 表记录已执行迁移。
   - 支持按文件名顺序执行 `*.up.sql`。
   - 每个迁移文件在独立事务中执行。

3. 完成 `cmd/migrate` 命令。
   - 默认执行 `up`。
   - 默认查找 `services/api/migrations` 或 `migrations` 目录。
   - 启动时读取 `.env` 和 `DATABASE_URL`。
   - 可通过 `-dir` 和 `-timeout` 调整迁移目录和超时时间。

4. 接入 API 数据库连接。
   - API 启动时读取 `DATABASE_URL`。
   - 启动阶段执行数据库 ping，失败则 fail fast。
   - `/health` 现在会检查数据库连通性。

5. 新增快捷命令。
   - `pnpm migrate:api`
   - `make migrate-api`
