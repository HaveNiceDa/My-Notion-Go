# 2026-05-12 22:43:00 进展记录

## 本次目标

实现 Auth MVP 后端主链路：注册、登录、JWT、Refresh Token、`GET /api/v1/me`。

## 已完成

1. 新增统一响应结构。
   - 新增 `internal/response/response.go`。
   - 统一返回 `success`、`data`、`error.code`、`error.message`。

2. 新增 Auth 数据模型与仓储。
   - 新增 `internal/auth/models.go`。
   - 新增 `internal/auth/repository.go`。
   - 覆盖 `users` 和 `refresh_tokens` 两张表的 GORM 模型。
   - 支持用户创建、按邮箱查询、按 ID 查询、Refresh Token 创建、查询和撤销。

3. 实现 Access Token JWT。
   - 新增 `internal/auth/token.go`。
   - 使用 HS256 方式签发和校验 JWT。
   - Access Token 包含 `sub`、`email`、`typ`、`exp`、`iat`。

4. 实现 Refresh Token。
   - 使用随机 32 字节 token。
   - 数据库存储 SHA-256 哈希，不保存明文 token。
   - 支持 refresh 时撤销旧 token 并签发新 token。
   - 支持 logout 撤销 refresh token。

5. 实现 Auth Service。
   - 新增 `internal/auth/service.go`。
   - 支持注册、登录、刷新 token、退出登录、获取当前用户。
   - 密码使用 `bcrypt` hash 和校验。
   - 注册时校验邮箱格式和密码长度。

6. 实现 Gin Handler。
   - 新增 `internal/auth/handler.go`。
   - 实现接口：
     - `POST /api/v1/auth/register`
     - `POST /api/v1/auth/login`
     - `POST /api/v1/auth/refresh`
     - `POST /api/v1/auth/logout`
     - `GET /api/v1/me`

7. 实现 JWT 鉴权中间件。
   - 新增 `internal/middleware/auth.go`。
   - 解析 `Authorization: Bearer <token>`。
   - 校验 Access Token 后把 `userID` 注入 Gin context。

8. 接入 API 路由。
   - 更新 `cmd/api/main.go`。
   - 将 Auth handler 接入 `/api/v1/auth`。
   - 将 `/api/v1/me` 作为受保护接口接入。

9. 更新配置。
   - 更新 `internal/config/config.go`。
   - 新增 `JWT_ACCESS_SECRET`、`JWT_REFRESH_SECRET`。
   - Access Token 默认 15 分钟过期。
   - Refresh Token 默认 30 天过期。

10. 更新 OpenAPI。
    - 更新 `services/api/docs/openapi.yaml`。
    - 补充 Auth MVP 接口定义和 Bearer JWT security scheme。
