# 2026-05-14 22:31:08 进展记录

## 本次目标

实现前端 Auth 最小闭环：登录、注册、token 存储、当前用户恢复和基础路由保护。

## 已完成

1. 扩展前端 API client。
   - 更新 `packages/api-client/src/index.ts`。
   - 新增统一响应解析，兼容后端 `success`、`data`、`error` 响应外壳。
   - 新增 `ApiError`，保留 HTTP status 和业务错误码。
   - 新增 Auth 类型：`User`、`TokenPair`、`AuthResult`、`RegisterRequest`、`LoginRequest`。
   - 新增 Auth API：注册、登录、刷新 token、退出登录、当前用户读取。

2. 新增 token 存储封装。
   - 新增 `apps/web/src/features/auth/authStorage.ts`。
   - 使用 `localStorage` 保存 MVP 阶段的 access token 和 refresh token。
   - 对外只暴露 `getTokens`、`setTokens`、`clearTokens`，方便后续替换为更安全的存储策略。

3. 实现前端 Auth 页面和登录态恢复。
   - 新增 `apps/web/src/features/auth/AuthShell.tsx`。
   - 支持 `/login`、`/register`、`/app` 三个入口。
   - 登录和注册成功后保存 token 并进入 `/app`。
   - 页面刷新时优先调用 `GET /api/v1/me` 恢复当前用户。
   - access token 失效时尝试使用 refresh token 轮换一组新 token。
   - refresh 失败时清理本地 token 并回到登录页。

4. 实现基础路由保护。
   - `/app` 作为受保护区域，未登录时自动回到 `/login`。
   - 已登录用户进入临时 Dashboard，展示用户 ID、邮箱和名称。
   - 支持退出登录，调用后端 logout 后清理本地 token。

5. 更新 Web 入口和样式。
   - 更新 `apps/web/src/app/App.tsx`，从占位首页切换为 `AuthShell`。
   - 更新 `apps/web/src/styles/global.css`，补充登录/注册页、受保护 Dashboard、表单和响应式布局样式。
