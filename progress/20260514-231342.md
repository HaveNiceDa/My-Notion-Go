# 2026-05-14 23:13:42 进展记录

## 本次目标

接入成熟前端基础库，避免 Web Auth 继续手写路由、全局状态、请求生命周期和常用 hook 逻辑。

## 已完成

1. 安装并接入前端基础库。
   - 为 `@my-notion-go/web` 安装 `react-router-dom`。
   - 安装 `react-hook-form`、`zod`、`@hookform/resolvers`。
   - 安装 `ahooks`。
   - `zustand` 原本已在 Web 依赖中，本次开始实际用于 Auth 全局状态。

2. 用 React Router 替换手写 History 路由。
   - 更新 `apps/web/src/main.tsx`，在根节点接入 `BrowserRouter`。
   - 更新 `apps/web/src/features/auth/AuthShell.tsx`，使用 `Routes`、`Route`、`Navigate`、`Outlet`、`Link`、`useNavigate`。
   - 保留 `/login`、`/register`、`/app` 三个入口。
   - 新增 public-only 和 protected route guard。

3. 用 Zustand 抽离 Auth 全局状态。
   - 新增 `apps/web/src/features/auth/authStore.ts`。
   - Auth 状态统一包含 `status`、`user`、`accessToken`、`refreshToken`。
   - 登录、注册、刷新 token 统一通过 `applyAuthResult` 落盘和更新状态。
   - 启动恢复登录态和退出登录逻辑从组件中移入 store。
   - 使用共享 `restorePromise` 避免 React StrictMode 下重复轮换 refresh token。

4. 用 React Hook Form + Zod 替换手写表单状态。
   - 登录和注册表单由 `useForm` 管理。
   - 邮箱和密码校验由 Zod schema 承担。
   - 表单错误统一走 `formState.errors` 和 `setError("root")`。

5. 用 ahooks 替换请求和常用 hook 逻辑。
   - 使用 `useMount` 执行启动登录态恢复。
   - 使用 `useRequest` 管理登录、注册和退出登录请求生命周期。
   - 使用 `useMemoizedFn` 管理提交和退出按钮的稳定回调。
   - 组件不再手写请求 loading/error 状态。

6. 创建项目 Web hooks Skill。
   - 新增 `.trae/skills/my-notion-go-web-hooks/SKILL.md`。
   - 约定 Web 端路由用 React Router，全局状态用 Zustand，表单用 React Hook Form + Zod。
   - 约定请求生命周期、稳定 callback、mount-only effect、防抖、节流、定时器等优先使用 ahooks。
