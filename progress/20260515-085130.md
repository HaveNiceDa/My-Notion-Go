# 2026-05-15 08:51:30 进展记录

## 本次目标

实现后端 Document 模块的 MVP，包括数据模型、兼容性 migration、Repository / Service / Handler 和受保护 API 路由。

## 已完成

1. 新增 Document 数据模型。
   - 新增 `services/api/internal/documents/models.go`。
   - 定义 `Document`，对应 `documents` 表。
   - 定义 `DocumentContent`，对应 `document_contents` 表。
   - 定义 `DocumentDTO` 和 `DocumentTreeDTO`，用于 API 响应和侧边栏树结构。

2. 新增 Document 数据访问层。
   - 新增 `services/api/internal/documents/repository.go`。
   - 支持创建文档时同步初始化空正文 `[]`。
   - 支持查询文档详情、文档树、回收站列表、更新元信息、归档/恢复子树和永久删除子树。
   - 使用 `user_id` 过滤所有查询，确保用户只能访问自己的文档。

3. 新增 Document 业务层。
   - 新增 `services/api/internal/documents/service.go`。
   - 支持默认标题 `Untitled`。
   - 支持按父子关系构建递归文档树。
   - 对文档 ID 和 parentId 做基础 UUID 格式校验，避免无效 ID 直接打到数据库形成 500。

4. 新增 Document HTTP Handler。
   - 新增 `services/api/internal/documents/handler.go`。
   - 新增接口：
     - `POST /api/v1/documents`
     - `GET /api/v1/documents/tree`
     - `GET /api/v1/documents/trash`
     - `GET /api/v1/documents/:id`
     - `PATCH /api/v1/documents/:id`
     - `POST /api/v1/documents/:id/archive`
     - `POST /api/v1/documents/:id/restore`
     - `DELETE /api/v1/documents/:id`

5. 注册 API 路由。
   - 更新 `services/api/cmd/api/main.go`。
   - 所有 Document 路由均使用 `middleware.RequireAuth(tokenManager)` 保护。

6. 补充兼容性 migration。
   - 新增 `services/api/migrations/000002_ensure_documents_schema.up.sql`。
   - 使用 `CREATE TABLE IF NOT EXISTS` 和 `CREATE INDEX IF NOT EXISTS`，兼容已经执行过早期 000001 的本地数据库。
   - 新增 `000002_ensure_documents_schema.down.sql`，不主动删除表，避免误删由 000001 创建的业务表。

7. 补充手动 API 调试文件。
   - 新增 `services/api/docs/documents.http`。
   - 覆盖登录、创建根文档、创建子文档、查询树、详情、更新、归档、回收站、恢复、永久删除。

8. 补充学习型 Go 注释。
