## 已完成

1. 实现后端 Content API。
   - 新增 `services/api/internal/documents/content_repository.go`。
   - 新增 `services/api/internal/documents/content_service.go`。
   - 新增 `services/api/internal/documents/content_handler.go`。

   - 路由已注册在 `services/api/cmd/api/main.go`。

2. 完善正文响应模型。
   - 更新 `services/api/internal/documents/models.go`。
   - 新增 `DocumentContentDTO`。
   - `content` 使用 `json.RawMessage` 返回，避免 Go 的 `[]byte` 被 JSON 编码成 base64 字符串。
   - 保留 `contentHash`、`version`、`updatedAt`，为后续冲突检测和同步调试预留字段。

3. 后端保存逻辑。
   - 保存前先校验 `user_id + document_id`，确保用户只能访问自己的文档正文。
   - 校验 `content` 必须是合法 JSON，并且顶层必须是数组，匹配 BlockNote 根 block 结构。
   - 保存时覆盖 `document_contents.content`。
   - 保存时生成 `sha256` content hash。
   - 保存时递增 `version`。

4. 更新 API 测试文件。
   - 更新 `services/api/docs/documents.http`。
   - 新增 `Get Document Content`。
   - 新增 `Update Document Content`。
   - 示例 JSON 使用 BlockNote-like paragraph block。

     - `@blocknote/mantine`
   - 在 `apps/web/src/main.tsx` 引入 `@blocknote/mantine/style.css`。

6. 扩展前端 API client。

7. 接入 React Query。

8. 实现自动保存。
   - 新增 `apps/web/src/features/documents/useAutosaveDocumentContent.ts`。
