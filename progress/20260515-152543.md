## 前端报错原因判断

1. 控制台错误：
   - `Uncaught RangeError: Invalid array passed to renderSpec`
   - 栈里落在 ProseMirror / BlockNote 初始化渲染阶段。

2. 触发点：
   - 文档详情页加载正文后，`DocumentEditor` 把后端返回的 `content` 直接作为 `initialContent` 传给 `useCreateBlockNote`。

3. 风险点：
   - 后端空正文是 `[]`。
   - 早期 `.http` 手动写入的 block 可能不是 BlockNote 当前版本完全认可的结构。
   - BlockNote 对 `initialContent` 要求较严格：必须是至少一个合法 block 的数组。

## 已修复

1. 更新 `apps/web/src/features/documents/DocumentEditor.tsx`。
   - 新增默认空段落：
     - `{ type: "paragraph" }`
   - 空数组不再直接传给 BlockNote，而是降级为默认空段落。
   - 非数组正文降级为默认空段落。
   - 历史异常 block 会经过基础清洗。
   - 只保留基本合法的 block。
   - 如果 `content` 不是数组或字符串，则删除该字段。
   - 如果 `children` 不是数组，则删除该字段。

2. 这样可以避免：
   - `[]` 空正文导致编辑器初始化异常。
   - 手动测试写入的非标准 block 直接传入 BlockNote。
   - 历史脏数据让整个文档详情页白屏。
