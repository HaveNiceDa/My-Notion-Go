# 2026-05-15 22:42:36 BlockNote renderSpec 根因修正

## 背景

- 文档详情页集成 BlockNote 后出现 `RangeError: Invalid array passed to renderSpec`。
- 最初通过运行时埋点确认 `renderSpec` 收到 `{ dom, contentDOM }` 结构，并用前端 shim 临时兜底。
- 用户指出官网最小 demo 不应需要该 shim，因此追加了纯官方最小用法页面做对照验证。

## 根因

- `/blocknote-demo` 只使用 `useCreateBlockNote()` 和 `BlockNoteView` 仍能复现同样错误。
- 这说明问题不来自后端保存、正文 JSON、autosave 或文档详情业务代码。
- 对比原 My-Notion 的 lockfile 后发现关键差异：
  - 原项目：`prosemirror-model@1.25.4`
  - 当前项目：`prosemirror-model@1.25.5`
- 将 `prosemirror-model` 对齐为 `1.25.4` 后，最小 demo 恢复正常。

## 修复

- 在根 `package.json` 的 `pnpm.overrides` 中增加：
  - `prosemirror-model: 1.25.4`
- 重新执行 `pnpm install --no-frozen-lockfile` 更新 `pnpm-lock.yaml`。
- 删除 `DocumentEditor.tsx` 中临时 `DOMSerializer.renderSpec` shim。
- 删除临时验证页面 `BlockNoteMinimalDemo.tsx`，并恢复 `App.tsx` 原路由入口。
- 移除 `apps/web/package.json` 中不再需要的直接依赖 `@tiptap/pm`。

## 验证

- 用户重启前端 dev server 后访问最小 demo，确认页面正常。
- `pnpm --filter @my-notion-go/web why prosemirror-model` 确认解析为 `1.25.4`。
- `pnpm --filter @my-notion-go/web typecheck` 通过。
- VS Code 诊断无类型错误，仅剩 `blocknote` 拼写提示。
