# 2026-05-15 22:58:59 编辑器体验与文档基础操作补齐

## 背景

- 用户验证发现 BlockNote 输入 `/` 后无法唤起默认富文本命令菜单。
- 用户希望文档基础操作在前端更完整体现，包括移动文档、重命名、在父文档下添加子文档。

## 前端调整

- 删除 `DocumentEditor.tsx` 中的 `formattingToolbar={false}`，恢复 BlockNote 默认选区格式工具栏。
- 之前已删除 `slashMenu={false}`，因此 `/` 命令菜单和默认 formatting toolbar 都恢复为 BlockNote 默认体验。
- 新增 `DocumentTreeActions.tsx`：
  - 添加子页面。
  - 重命名页面。
  - 通过下拉框移动页面到根层级或其他父页面。
- 新增 `documentTreeUtils.ts`，把文档树拍平成移动目标列表，并保留层级缩进信息。
- `DocumentTree` / `DocumentTreeItem` / `WorkspaceSidebar` / `DocumentWorkspace` 串联文档操作 mutation。
- 新增中英文 i18n 文案，覆盖重命名、移动、移动到根层级等 UI 文案。
- 补充文档树行内操作样式。

## 后端调整

- `PATCH /api/v1/documents/:id` 新增 `parentId` 支持：
  - `parentId: ""` 表示移动到根层级。
  - `parentId: "<documentId>"` 表示移动到目标父文档下。
- Service 层校验不能移动到自身。
- Repository 层在事务中移动文档：
  - 更新 `parent_id`。
  - 追加到新父级的末尾 position。
  - 更新当前文档 path。
  - 同步更新所有后代文档 path。
  - 阻止把文档移动到自身后代中，避免形成循环树。
- `documents.http` 增加移动到根层级和移动回父文档的手动验证示例。
