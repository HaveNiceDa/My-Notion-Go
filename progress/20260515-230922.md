# 2026-05-15 23:09:22 文档树交互对齐 My-Notion

## 背景

- 用户反馈文档树重命名图标不好看、位置不对。
- 用户要求重命名弹窗不要使用原生 `prompt`，改用 shadcn 风格组件。
- 用户期望移动文档参考原 My-Notion，通过拖拽文档项完成，而不是下拉框选择。

## 参考

- 对照原 My-Notion：
  - `apps/web/src/app/[locale]/(main)/_components/Item.tsx`
  - `apps/web/src/components/modals/rename-modal.tsx`
  - `apps/web/src/components/ui/dialog.tsx`
  - `apps/web/src/components/ui/dropdown-menu.tsx`

## 实现

- 新增本地 shadcn/Radix 风格基础组件：
  - `apps/web/src/components/ui/dialog.tsx`
  - `apps/web/src/components/ui/dropdown-menu.tsx`
- 新增 `RenameDocumentDialog.tsx`，替代 `window.prompt`。
- `DocumentTreeActions` 改为 My-Notion 风格：
  - hover 后显示 `MoreHorizontal`。
  - 重命名入口放进 dropdown menu。
  - 子页面创建保留为右侧 hover `Plus`。
- 删除移动下拉框，改为拖拽移动：
  - 拖动文档项到另一个文档项上，会调用后端 `parentId` 移动接口。
  - 拖拽时显示透明态和 drop target 高亮。
- 更新 CSS，补充 dropdown、dialog、拖拽态和行内 action 样式。
- 更新 i18n，补充 dialog、更多操作、保存/取消等文案。
- 更新 `.trae/skills`：
  - 禁止产品 UI 使用原生 `prompt/alert/confirm`。
  - 要求弹窗/菜单使用 shadcn/Radix 风格组件。
  - 文档树操作对齐原 My-Notion 的 `MoreHorizontal + Plus + drag/drop` 模式。
