# 2026-05-16 09:57:59 文档树拖拽移动修复

## 背景

- 用户反馈 `my-notion-go` 文档移动不可用，并要求参考原 My-Notion 的实现。
- 原 My-Notion 使用 HTML5 原生拖拽：
  - 文档行设置 `draggable`。
  - `dragstart` 将源文档 ID 写入 `dataTransfer`。
  - `drop` 到目标文档行后调用移动接口，把源文档挂到目标文档下。

## 根因

- `my-notion-go` 前端在上次收口时删除了拖拽移动入口，只保留了重命名和新增子页面操作。
- 后端 `PATCH /api/v1/documents/:id` 的 `parentId` 移动能力仍然存在，但前端没有再调用。
- 后端 `buildTree` 旧实现提前把子节点按值 append 到父节点，可能导致三层及以上文档树丢失孙级节点展示。

## 修复

- `DocumentTreeItem.tsx` 恢复 HTML5 原生拖拽：
  - 拖拽文档行时写入源文档 ID。
  - drop 到目标文档行时调用 `onMove(sourceId, targetId)`。
  - 移动成功后展开目标节点，便于看到新子页面。
  - 保留 `Link draggable={false}`，避免拖动链接触发浏览器拆分视图。
- `DocumentTree.tsx`、`WorkspaceSidebar.tsx`、`DocumentWorkspace.tsx` 串联 `onMove`。
- `DocumentWorkspace.tsx` 复用现有 `updateDocument` mutation，提交 `{ parentId }` 到后端。
- `global.css` 补充拖拽中和 drop target 高亮样式。
- `service.go` 重写 `buildTree`，通过 parent -> child id 索引递归生成树，避免值拷贝导致深层节点丢失。
- 新增 `service_test.go`，覆盖根节点、子节点、孙节点的树组装回归测试。
