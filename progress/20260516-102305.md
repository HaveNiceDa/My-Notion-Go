# 2026-05-16 10:23:05 shadcn 基础设施接入

## 背景

- 用户确认 `my-notion-go` 应正式采用 shadcn/ui 组件体系。
- 原 My-Notion 大量使用 shadcn 风格组件，且 shadcn 组件源码可复制到项目内继续自定义。
- `my-notion-go` 之前只有手写的 Radix `Dialog` 和 `DropdownMenu` 封装，没有 Tailwind、`components.json`、`cn` 工具和路径别名。

## 实现

- 接入 Tailwind CSS v3、PostCSS、`tailwindcss-animate`。
- 新增 `components.json`，声明 shadcn aliases 和 Tailwind 入口。
- 新增 `tailwind.config.cjs` 和 `postcss.config.cjs`。
- 新增 `src/lib/utils.ts`，提供 shadcn 标准 `cn()` 工具。
- 配置 `@/*` TypeScript path alias 和 Vite alias。
- 新增 shadcn 风格组件：
  - `components/ui/button.tsx`
  - `components/ui/input.tsx`
- 将现有 Radix 封装迁移为 shadcn 风格：
  - `components/ui/dialog.tsx`
  - `components/ui/dropdown-menu.tsx`
- `RenameDocumentDialog.tsx` 开始使用新的 `Button`、`Input`、`Dialog` 组件。

## 说明

- 当前只完成 shadcn 基础设施和首批组件迁移，不做全量 UI 重构。
- Tailwind 颜色使用现有 CSS 变量，避免破坏当前 Notion-like 视觉。
- 后续可以逐步迁移 `AuthDialog`、导航按钮、搜索、设置和 AI Chat 相关 UI。

## 验证

- `pnpm --filter @my-notion-go/web typecheck` 通过。
- `pnpm --filter @my-notion-go/web build` 通过。
- Vite build 仍有大 chunk 提示，来自 BlockNote 等编辑器依赖，不影响本次 shadcn 接入。
