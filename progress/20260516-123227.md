# 2026-05-16 12:32:27 M4.1 OpenAI Compatible SSE 接入

## 背景

- M4 已完成 AI Chat mock SSE 的后端和前端最小闭环。
- 用户希望接入真实 OpenAI Compatible API，但不确定 Go 侧如何调用现有 AI SDK/环境变量。

## 实现

- 新增 `services/api/internal/ai` 模块：
  - 使用 Go 标准库 `net/http` 调用 OpenAI Compatible `/chat/completions`。
  - 请求体使用 `stream: true`。
  - 解析 `text/event-stream` 的 `data:` frame。
  - 支持 `[DONE]` 结束标记。
  - 将 `choices[].delta.content` 转发为内部 delta。
- 扩展后端配置：
  - `LLM_API_KEY`
  - `LLM_BASE_URL`
  - 兼容 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`DASHSCOPE_API_KEY`。
- 更新 AI Chat stream：
  - 有可用 LLM 配置时调用真实 OpenAI Compatible stream。
  - 未配置 API key 时保留 mock fallback，避免本地开发和 smoke 测试强依赖外部模型服务。
  - 前端 SSE 事件协议保持不变：`conversation`、`user_message`、`message`、`assistant_message`、`done`。
  - assistant 消息 metadata 记录 provider/model 或 mock provider。
- 更新 AI Chat smoke：
  - 不再强依赖 assistant 内容包含 `Mock AI response`。
  - 改为验证 assistant 消息内容非空，兼容真实 LLM 和 mock fallback。
