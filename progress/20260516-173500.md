# 2026-05-16 17:35:00 M5.1 RAG 数据模型与 API 骨架

## 背景

- M5.0 已完成 DashScope 原生 embedding、Qdrant 初始化和 embedding/Qdrant smoke。
- M5.1 目标是先落 PostgreSQL 状态模型和 RAG API 骨架，暂不执行真实切块与向量写入。
- 产品规则调整为：每篇文档默认进入知识库，用户可以显式关闭。

## 实现

- `services/api/migrations/000004_rag_schema.up.sql`：
  - 新增 `rag_documents`，记录单篇文档的 RAG 索引状态。
  - 新增 `rag_chunks`，为 M5.2 的 chunk 元数据和 Qdrant point id 做准备。
  - 新增 `jobs`，为后续 worker/RabbitMQ 异步索引做准备。
  - 将 `documents.is_in_knowledge_base` 默认值改为 `true`。
  - 将历史未删除文档迁移为默认进入知识库。
- `services/api/internal/documents`：
  - 新文档创建时显式设置 `IsInKnowledgeBase=true`。
  - GORM model 默认值同步为 `default:true`。
- `services/api/internal/rag`：
  - 新增 `models.go`、`repository.go`、`service.go`、`handler.go`。
  - `POST /rag/documents/:id/index`：开启/重新开启知识库，状态置为 `pending`。
  - `DELETE /rag/documents/:id/index`：关闭知识库，状态置为 `disabled`。
  - `GET /rag/documents/:id/status`：查询状态；默认开启但无记录时自动创建 `pending`。
- `services/api/docs/rag.http`：
  - 新增手动调试请求。
- `scripts/smoke-rag-api.mjs`：
  - 新增 `pnpm smoke:api:rag`。
  - 覆盖默认开启、关闭、状态查询、重新开启和清理流程。
