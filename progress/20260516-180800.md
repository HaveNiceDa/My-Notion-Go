# 2026-05-16 18:08:00 M5.2 文档切块与真实索引

## 背景

- M5.1 已完成 RAG 状态表和 API 骨架。
- M5.2 目标是把 `POST /api/v1/rag/documents/:id/index` 从状态切换升级为真实索引闭环。
- 当前阶段仍采用同步索引，方便 smoke 验证；后续再迁移到 worker/RabbitMQ。

## 实现

- `services/api/internal/rag/chunker.go`：
  - 实现 BlockNote JSONB 到纯文本的最小转换。
  - 支持递归读取 block children。
  - 支持 inline text 数组和 string content。
  - 采用固定字符窗口切块，并保留少量 overlap。
  - 保留来源 block ids，后续用于引用回跳。
- `services/api/internal/rag/repository.go`：
  - 新增 `CreateChunks`。
  - 新增 `DeleteChunksByDocumentID`。
  - 新增 `ListPointIDsByDocumentID`。
  - 新增 `MarkFailed`。
  - `UpsertDocumentStatus` 在 pending/disabled/indexing 时会重置旧 `chunk_count`、`content_hash` 和 `indexed_at`。
- `services/api/internal/rag/qdrant.go`：
  - 新增 `UpsertPoints`。
  - 新增 `DeletePoints`。
  - point id 由 `rag_chunks.qdrant_point_id` 持久化，便于重建和关闭知识库时精准删除。
- `services/api/internal/rag/service.go`：
  - `EnableDocumentIndex` 改为同步执行索引。
  - 索引流程：读取文档内容 -> BlockNote 转纯文本 -> chunk -> DashScope embedding -> 写 `rag_chunks` -> Qdrant upsert -> 标记 `indexed`。
  - `DisableDocumentIndex` 会删除 Qdrant points 和 `rag_chunks`，再标记 `disabled`。
  - 索引失败会写入 `failed` 和 `last_error`，方便后续前端提示。
- `scripts/smoke-rag-api.mjs`：
  - 写入一段 BlockNote 内容。
  - 验证索引后状态为 `indexed` 且 `chunkCount > 0`。
  - 验证关闭知识库会清空 chunk。
  - 验证重新开启会重建索引。

## 注意

- 当前 chunker 是最小实现，不完整理解 BlockNote 所有复杂 block。
- 当前索引是同步执行，适合 M5.2 验证闭环；大文档会在 M5.3/M5.4 之后迁移到异步 worker。
- Qdrant search 和 RAG Chat SSE 尚未实现。
