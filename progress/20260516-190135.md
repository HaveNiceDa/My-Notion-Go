# 2026-05-16 19:01 M5.4 RAG Knowledge Base Fix

## 背景

- 前端 RAG 模式已接入 `/api/v1/rag/chat/stream`，但不同用户索引数据不一致时，用户会看到 404。
- 需要把 RAG 从“后端能力可用”推进到“用户可感知、可操作、可降级”的产品体验。

## 实现内容

- RAG Chat 降级：
  - 无索引或无有效检索结果时，不再返回 `RAG_CONTEXT_NOT_FOUND`。
  - 继续复用 AI Chat 流式回答，assistant metadata 写入 `rag.enabled=false`、`rag.fallback=true`。
- 用户维度过滤：
  - Qdrant search 仍使用 `userId` filter。
  - 命中结果再通过 PostgreSQL 校验 `rag_chunks`、`documents.is_in_knowledge_base`、`rag_documents.status=indexed`。
  - 过滤掉 stale point、关闭知识库文档和非 indexed chunk。
- 自动索引：
  - `documents.Service` 增加内容保存 hook。
  - `PUT /api/v1/documents/:id/content` 保存成功后后台触发当前文档重建索引。
  - hook 注入放在 `main.go`，避免 `documents` 包反向 import `rag`。
- 文档页入口：
  - API client 增加 RAG status、enable、disable 方法。
  - 文档顶部栏增加知识库开关。
  - 新增轻量 toast provider，开启/关闭知识库时给用户反馈。
- Smoke 调整：
  - 正文保存后可能被自动索引，`smoke:api:rag` 初始状态兼容 `pending/indexing/indexed`。
