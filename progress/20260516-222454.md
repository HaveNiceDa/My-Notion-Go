# 2026-05-16 22:24:54 M5.4 RAG 与 AI 面板体验收口

## 背景

- M5.4 已完成 RAG 前端接入，但引用展示、AI 入口、知识库开关和编辑器离开页面保存体验还需要收口。
- 用户确认暂不处理敏感信息索引保护，优先优化当前 Notion-like 使用体验和 RAG 命中可观测性。

## 实现

- `AIChatPanel`：
  - RAG citation 改为新标签页打开来源文档，避免替换当前工作区页面。
  - 等待后端首个 SSE delta 时增加“AI 正在思考”loading 气泡。
  - 顶部布局改为 Notion 风格紧凑栏：左侧会话标题和历史下拉，右侧新会话与关闭按钮。
  - 模型选择移动到底部输入框内。
  - 知识库模式从下拉选择改成底部单按钮开关，默认普通对话，点击后 toast 提示开启/关闭。
  - 历史会话列表去掉每条前面的对话 icon，文字颜色加深。
  - RAG citation 展示相似度分数，preview 改为单行省略。
  - citation 标题优先使用后端返回的真实 `documentTitle`，缺失时兜底为“未命名文档”，不再显示“来源文档 xx”。
- `internal/rag`：
  - RAG 检索增加 `minScore=0.6` 阈值过滤，低于阈值的 Qdrant 命中不会进入 RAG context 或 citations。
  - citation DTO 已包含 `documentTitle` 与 `score`，用于前端展示真实来源标题和相似度。
- `DocumentWorkspace`：
  - 新增右下角悬浮 AI 助手入口，点击可打开 AI 侧边栏，侧边栏打开时隐藏悬浮按钮。
  - 知识库开启/关闭请求增加 `keepalive`，用户点击后马上退出时浏览器会尽量把请求送达后端。
  - 删除不再展示的 RAG status query 轮询。
- `DocumentNavbar`：
  - 顶部知识库开关精简为单一 `知识库` pill。
  - 文档默认知识库开启；关闭后按钮置灰，再次点击可重新开启。
  - 不再在顶部展示 `pending/indexing/indexed/failed/disabled` 或 chunk 数，避免信息密度过高。
- `useAutosaveDocumentContent`：
  - 组件卸载时取消 debounce，并用 `keepalive` 发最后一次正文保存请求，降低切换文档或关闭页面时丢内容的风险。
  - 清理过期的 RAG status cache 更新、`ragDocumentStatusQueryKey` invalidate 和“轮询校准”注释。
- `components/ui/button.tsx`：
  - 在本地 shadcn `Button` primitive 统一加入 `cursor-pointer`，避免业务代码重复添加。
- `resources.ts`：
  - 补齐 citation score、知识库开关和 AI 面板相关中英文文案。
  - 精简知识库 toast description 为“加入知识库 / 移除知识库”。
  - 删除不再使用的 `knowledgeBaseStatus` 文案。

## 设计决策

- citation 点击使用新标签页打开，保留当前编辑上下文。
- RAG 相似度阈值先固定为 `0.6`，后续可根据 smoke 和真实数据再调参。
- 顶部知识库入口只表达开关状态，不再承载索引进度；索引仍由后端在保存或开关后处理。
- 知识库切换请求前端只需提交到后端并 toast，不等待索引完成。
- 卸载保存使用 `keepalive` 做兜底；正常编辑仍走原来的 debounce autosave。
