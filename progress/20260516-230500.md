# 2026-05-16 23:05:00 M5.4 RAG 前端引用与索引状态

## 背景

- M5.3 已完成 RAG SSE、citations metadata、Qdrant userId filter 和 PostgreSQL 二次校验。
- M5.4 继续补齐前端体验：RAG 引用需要能跳转来源文档，顶部栏需要展示知识库索引状态。

## 实现

- `AIChatPanel`：
  - RAG citations 从纯文本预览升级为可点击的 shadcn `Button`。
  - 点击引用后通过 React Router 跳转到 `/documents/:documentId`。
  - 引用项展示来源文档标题，缺失时使用兜底标题。
- `internal/rag`：
  - `CitationDTO` 新增 `documentTitle` 字段。
  - Qdrant 命中后继续通过 PostgreSQL 二次校验，并在 join `documents` 时带出文档标题。
  - `buildRAGContext` 保持原有 prompt，不把标题作为安全边界，只作为前端展示字段。
- `DocumentNavbar`：
  - 知识库开关旁展示 `pending`、`indexing`、`indexed`、`failed`、`disabled` 状态。
  - `indexed` 状态展示 chunk 数。
  - `failed` 状态使用 danger 色，并把 `lastError` 放入 title 便于定位。
- `DocumentWorkspace`：
  - RAG status query 在 `pending/indexing` 时每 2 秒短轮询，状态稳定后停止。
- `useAutosaveDocumentContent`：
  - 正文保存成功后，如果文档在知识库中，前端乐观标记 RAG 状态为 `indexing`。
  - 同时 invalidate RAG status query，由后端真实状态校准为 `indexed/failed`。
- `resources.ts`：
  - 补齐知识库状态和引用跳转的 `zh/en` 文案。
