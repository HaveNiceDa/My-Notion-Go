# 2026-05-17 15:10 M5.5 RAG 异步索引 Worker

## 背景

- M5.4 已完成 RAG 最小闭环、AI 面板入口、引用展示和自动索引触发。
- 剩余主要工程缺口是：正文保存和手动开启知识库仍会在 API 进程内直接触发索引执行，`jobs` 表和 `cmd/worker` 还没有真实参与任务流。
- 本阶段目标是先完成最小可靠的 worker abstraction，不立即引入 RabbitMQ 客户端依赖，避免基础设施复杂度阻塞现有 RAG 闭环。

## 实现

- 新增 `services/api/internal/jobs`：
  - `models.go` 定义通用 `Job`、任务状态和 `rag.index` 类型。
  - `repository.go` 实现 DB-backed enqueue、claim、mark completed/failed。
  - `EnqueueRAGIndex` 会合并同一用户同一文档的 pending 任务，避免 autosave 高频保存导致任务堆积。
  - running 任务不复用；如果索引执行期间用户继续保存正文，会保留新的 pending 任务处理最新内容。
- 改造 `internal/rag.Service`：
  - 新增 `SetJobRepository` 注入 jobs repository。
  - `EnableDocumentIndex` 在注入 jobs 后改为创建 `rag.index` job 并返回 pending/indexing 状态；未注入 jobs 时保留同步索引兜底。
  - `ReindexDocumentIfEnabled` 从“后台 goroutine 执行索引”改为“后台 goroutine 投递 job”，避免 autosave 请求被 embedding/Qdrant 慢调用拖住。
  - 新增 `ExecuteIndexJob` 供 worker 执行真实索引，并在文档已关闭知识库时安全地落为 `disabled`。
- 新增 `internal/rag.IndexWorker`：
  - 使用 PostgreSQL `FOR UPDATE SKIP LOCKED` 领取 pending job。
  - 执行切块、DashScope embedding、Qdrant upsert 和 `rag_documents` 状态更新。
  - 成功后写入 job result；失败后写入 job last_error，同时保留 RAG 状态中的失败信息。
- 改造 `cmd/worker`：
  - 读取 `.env`，连接 PostgreSQL，初始化 embedding client 和 Qdrant client。
  - 组装 `documents.Repository`、`rag.Repository`、`jobs.Repository` 和 `rag.IndexWorker`。
  - 支持 `SIGINT/SIGTERM` 优雅停止。
- 更新根脚本：
  - `pnpm dev` 同时启动 web/api/worker。
  - 新增 `pnpm build:worker`。
  - `pnpm build` 现在包含 worker 构建。
- 更新 `scripts/smoke-rag-api.mjs`：
  - `POST /rag/documents/:id/index` 兼容 `pending/indexing/indexed`。
  - 增加 `waitForIndexed` 轮询，等待 worker 完成异步索引。
  - 超时提示用户确认 `pnpm dev:worker` 正在运行。

## 设计决策

- 暂不接入 RabbitMQ：当前先使用 PostgreSQL jobs 表实现可观测、可测试的 worker abstraction，后续可把 `ClaimNext` 替换为 MQ consume。
- 不新增 Redis lock：pending 合并已能控制 autosave 高频任务数量；真正的跨进程互斥由 `FOR UPDATE SKIP LOCKED` 保证。
- API 不再等待索引完成：前端仍保持轻量知识库开关，索引最终状态由 `rag_documents` 和 smoke 轮询确认。
