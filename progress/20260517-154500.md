# 2026-05-17 15:45 Agent + Tool 方向规划

## 背景

- M5 已完成 RAG 最小闭环、引用展示、相似度过滤和异步索引 worker。
- 原计划中曾考虑继续做 RAG query intent 人工降级，避免普通问题触发知识库检索。
- 用户提出新的产品和架构判断：后续 AI 对话会演进为 Agent，RAG 只是 Agent 的一个 tool，后续还会接入联网搜索等更多 tool 能力。

## 决策

- 不继续推进人工 query intent 前置校验。
- 不把“是否需要 RAG”写死为后端启发式规则。
- RAG 后续作为 Agent 的 `knowledge_base.search` tool，由 Agent 根据上下文和模型规划自主决定是否调用。
- 联网搜索、文档操作、任务创建等能力也应以 tool 形式接入统一 Agent。
- 当前 `/api/v1/rag/chat/stream` 保留为过渡入口、调试入口和 smoke 验证入口；未来统一 Agent 可复用底层 RAG search 能力。
