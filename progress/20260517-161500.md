# 2026-05-17 16:15 M5.7 Citation 来源定位与高亮

## 背景

- M5.4 已在 AI 面板展示 RAG citations，并支持点击打开来源文档。
- M5.5 已把 RAG 索引迁移到异步 worker。
- Agent + Tool 方向确认后，RAG citation 需要保留更明确的 tool result 元数据，方便当前 UI 和未来 Agent 复用。

## 实现

- 后端 citation 元数据：
  - `CitationDTO` 新增 `blockIds` 字段。
  - `SearchResult` 新增 `BlockIDs`。
  - `ListEnabledChunksByIDs` 从 `rag_chunks.block_ids` 解析来源 BlockNote block ids。
  - `buildRAGContext` 在 citations 中带出 `blockIds`，SSE `citations` 事件和 assistant metadata 都会持久化这份定位信息。
- AI 面板：
  - citation 链接从 `/documents/:documentId` 升级为携带 query 参数：
    - `citationChunkId`
    - `citationBlockId`
    - `citationPosition`
  - 链接仍然使用新标签页打开，避免打断当前编辑上下文。
- 文档工作区：
  - `DocumentWorkspace` 通过 `useSearchParams` 解析 citation query。
  - `DocumentDetail` 将 citation target 传入 `DocumentEditor`。
- BlockNote 编辑器：
  - `DocumentEditor` 新增 `CitationHighlightTarget` 类型。
  - 编辑器挂载后使用 BlockNote DOM 的 `data-id` 定位来源 block。
  - 找到来源 block 后滚动到视口中部，并使用 Web Animations API 做短暂高亮。
  - 如果缺少 block id，则按 chunk position 尝试兜底定位 `.bn-block-outer`。
- Smoke：
  - `scripts/smoke-rag-api.mjs` 新增 `blockIds` 断言，确认 citation 能定位到 smoke 写入的 `rag-smoke-block-1`。
