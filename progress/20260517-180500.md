# 2026-05-17 18:05 M6.0 Search + Command Palette

## 背景

- M5 RAG + Worker + Qdrant 已收口，下一阶段进入 M6 工作区产品能力补齐。
- 用户询问搜索是否可以接入 Elasticsearch。

## 决策

- M6.0 搜索先采用 PostgreSQL 实现最小闭环，不立即引入 Elasticsearch。
- 理由：
  - 当前项目已有 PostgreSQL，能更快完成 Command Palette 搜索跳转。
  - 可以直接保证所有查询带 `user_id` 条件。
  - 避免过早引入 ES 集群、索引同步、删除清理和部署复杂度。
- API 返回独立的 `DocumentSearchResult`，后续可在后端替换为 ES adapter，前端协议保持稳定。

## 本次改动

- 新增 `GET /api/v1/documents/search`：
  - 支持 `q`、`limit`、`includeArchived`。
  - 标题命中优先，正文 JSONB 文本匹配兜底。
  - 默认不搜索归档文档。
- 前端新增轻量 Command Palette：
  - 侧边栏搜索按钮打开。
  - 支持 `Cmd/Ctrl + K`。
  - 搜索结果点击后跳转文档。
- `pnpm smoke:api:documents` 增加标题搜索和正文搜索断言。
- 新增 `milestones/M6-search-workspace-polish.md`，记录 M6 范围和 ES 取舍。
