# 2026-05-19 09:35 M8.2/M8.3 Mobile Read + Metadata Editor

- M8 已进入原生移动端阶段，`apps/mobile` 使用 Expo Router + React Native + NativeWind。
- M8.1 已完成移动端鉴权、SecureStore refresh token、内存态 access token 和 `runWithAuth` 401 静默刷新。

## 当前问题

- 移动端功能闭环已推进到 M8.3，但视觉质量明显不足：
  - 当前 `my-notion-go` 移动端更像 demo 卡片堆叠，缺少 Notion 移动端的低对比、紧凑列表和原生导航感。
  - 页面大量使用大圆角、大卡片、强阴影和粗重标题，和原 My-Notion 移动端以及 Notion 原生 App 的信息密度不一致。
  - 文档列表、详情、搜索、回收站和编辑卡片缺少统一的 spacing、row、section header、bottom action 等组件规范。
  - 当前配色 `#f7f5f2`、`#ede9e3` 偏暖偏重，和 Web 端 `#ffffff` / `#f7f7f5` / `#ededeb` / `#e5e5e2` 的 Notion-like 色系不完全一致。

## 样式重构决策

- 在进入 M8.4 AI + RAG Mobile 前，新增一个阶段：`M8.3.5 Mobile Visual Refactor`。
- 目标不是只调几个颜色，而是建立一套移动端 Notion-like 视觉系统：
  - 参考原 My-Notion 移动端的 workspace header、recent 横向卡片、折叠分组、树形 row、底部操作栏。
  - 参考 `my-notion-go` Web 端的 Notion-like token，包括白底、浅灰 hover、弱边框、黑白主按钮和低对比文字。
  - 参考公开可见的 Notion 原生移动端风格，但不复制不可获得的私有设计 token。
- 重构应优先沉淀 `Screen`、`Header`、`Section`、`Row`、`IconTile`、`BottomActionBar`、`ActionPill` 等基础组件，再迁移业务页面。

## 后续

- 优先执行 M8.3.5，而不是直接进入 M8.4。
- M8.3.5 完成后再做 AI 移动端入口，避免 AI 页面继续继承当前粗糙视觉基线。
